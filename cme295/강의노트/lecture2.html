01 — Overview

이 강의를 관통하는 질문: 트랜스포머는 '위치'와 '비용'을 어떻게 다루는가? Position & Efficiency in Transformers

대규모 동창회장에 처음 들어섰다고 상상해 봅시다. 명찰이 없으면 누가 몇 번 테이블에 앉았는지조차 알 수 없고, 참석자 한 명 한 명과 눈을 맞추며 대화하려면 시간과 체력이 금세 바닥납니다. 지난 강의에서 배운 어텐션(Attention)도 똑같은 처지에 있습니다.

토큰들 사이의 직접 연결만으로는 순서 정보를 잃어버리고, 모든 토큰이 서로를 전부 들여다보는 방식은 문장이 길어질수록 감당하기 어려워집니다. 결론부터 말하면, 이번 2강은 이 두 가지 현실적 문제 — "자리를 어떻게 표시할까"와 "대화 비용을 어떻게 줄일까" — 에 대한 트랜스포머(Transformer) 생태계의 해법을 훑습니다.

먼저 위치 인코딩(Positional Encoding)이 학습형에서 사인 함수 기반, 그리고 회전 방식인 RoPE(Rotary Position Embedding, 회전 위치 임베딩)로 진화한 과정을 살펴보겠습니다. 그다음 층마다 신호 크기를 다시 맞추는 레이어 정규화(Layer Normalization)의 위치가 왜 학습 안정성을 좌우하는지 보고, Longformer의 희소 어텐션(Sparse Attention)부터 다중 쿼리 어텐션(Multi-Query Attention, MQA)·그룹 쿼리 어텐션(Grouped-Query Attention, GQA)까지 이어지는 어텐션 근사(Attention Approximation) 계보를 따라갑니다. 마지막으로 인코더만 남긴 BERT(Bidirectional Encoder Representations from Transformers, 트랜스포머의 양방향 인코더 표현)가 어떻게 사전학습-파인튜닝 패러다임을 열었는지 정리합니다.

원본 슬라이드 12쪽의 목차
강의는 위치 임베딩(Position embeddings) → 레이어 정규화(Layer normalization) → 어텐션 근사(Attention approximation) → 트랜스포머 기반 모델(Transformer-based models) → BERT 심화(BERT deep dive) 순서로 진행됩니다. 이 페이지도 같은 순서를 따릅니다.
①

자리를 어떻게 표시할까

직접 연결만으로는 위치가 사라집니다. 학습형 → 고정 사인함수 → 상대편향(ALiBi) → 회전(RoPE)까지, 위치 정보를 주입하는 방법이 진화해 온 이유를 봅니다.

②

대화 비용을 어떻게 줄일까

모두가 모두를 쳐다보는 MHA(Multi-Head Attention, 다중 헤드 어텐션)는 비쌉니다. 희소 어텐션과 K/V 헤드 공유(MQA·GQA)로 비용을 낮추는 전략을 봅니다.

③

인코더만 남기면 무엇이 좋을까

디코더를 떼어내고 양방향 문맥만 취한 BERT가 왜 분류·이해 과제에서 강력했는지, 그리고 어떤 한계 때문에 지금은 디코더-온리 모델이 더 널리 쓰이는지 살펴봅니다.

02 — Definitions & Mechanism

왜 위치 정보가 필요할까? From Absolute to Rotary Position

회의 테이블 위 명찰이 없다면 옆자리 사람이 누구인지 알 길이 없습니다. 어텐션도 마찬가지로, 쿼리(Query)·키(Key)·값(Value) 사이의 직접 연결만으로는 "teddy bear"가 문장의 세 번째 단어인지 여섯 번째 단어인지 구별하지 못합니다.

원본 동기: 직접 연결은 위치 정보를 "잃는다"
쿼리·키·값이 토큰 임베딩에서 바로 계산되기 때문에, 문장 속에서 단어의 순서를 바꿔도 어텐션 계산 자체는 달라지지 않습니다. "Direct links 'lose' position info"가 이 강의가 위치 인코딩에서 출발하는 이유입니다.

1) 학습형 위치 임베딩과 그 한계

가장 단순한 해법은 학습형 위치 임베딩(Learned Positional Embedding)입니다. 자리마다 고유한 명찰을 인쇄해 나눠주듯, 위치 0, 1, 2, …마다 학습 가능한 벡터를 하나씩 배정하고 토큰 임베딩에 더합니다. 문제는 원본 훈련 때 본 적 없는 위치(예: 훈련은 512 토큰까지만 했는데 추론은 1000 토큰)에는 명찰 자체가 존재하지 않는다는 점입니다 — "Need to retrain for longer sequences".

2) 고정된 사인 함수 위치 인코딩

원본 Vaswani et al.(2017)의 원조 트랜스포머는 대신 사인·코사인 함수로 값을 하드코딩합니다. 마치 시계 여러 개를 서로 다른 속도로 돌려 놓고, 각 시곗바늘의 각도 조합으로 "지금 몇 시인지"를 유일하게 표현하는 것과 같습니다. 차원 쌍마다 회전 속도(주파수)가 다르기 때문에, 짧은 주기의 바늘은 인접 위치를 세밀하게 구별하고 긴 주기의 바늘은 먼 위치까지 구별해 줍니다.

PEm,2i = sin( m / 100002i∕dmodel )　　PEm,2i+1 = cos( m / 100002i∕dmodel ) 위치 m, 차원 인덱스 i. 짝수 차원엔 sin, 홀수 차원엔 cos을 채워 각 차원 쌍이 서로 다른 속도로 회전하는 "시곗바늘"이 됩니다.

핵심은 삼각함수 항등식 원본 cos(a−b) = cos(a)cos(b) + sin(a)sin(b)을 이용하면 두 위치 임베딩의 내적이 오직 위치 차이(m−n)만의 함수가 된다는 사실입니다.

⟨PEm, PEn⟩ = f(m − n) 절대 위치 m, n이 아니라 상대 거리 m−n만 남는다는 뜻입니다. 이 성질이 아래 RoPE로 이어지는 다리가 됩니다.

원본 이 방식의 장점은 벡터를 다시 훈련하지 않아도 임의의 시퀀스 길이로 확장할 수 있다는 것입니다.

위젯 · 사인 함수 위치 인코딩 히트맵 종합·해석

3) 관점을 바꾸다: 절대 위치 대신 상대 위치를 어텐션 층에 넣기

원본 "Actually, we really care about relative position with attention layer" — 애초에 우리가 궁금한 건 "teddy bear"의 절대 좌표가 아니라 "teddy bear가 cute보다 몇 칸 뒤에 있는가"입니다. 그래서 위치 정보를 토큰 벡터에 더하는 대신, 아예 어텐션 계산 자체를 바꾸는 접근이 등장합니다.

학습형 절대위치 원본

위치마다 학습 가능한 임베딩 벡터를 하나씩 두고 토큰 임베딩에 더합니다. 구현이 가장 단순하고 데이터로부터 최적의 위치 표현을 배울 수 있지만, 훈련 때 보지 못한 위치로는 확장할 수 없습니다(재훈련 필요). BERT를 포함한 여러 초기 트랜스포머가 이 방식을 사용했습니다.

고정 사인함수(Sinusoidal) 원본

학습 파라미터 없이 sin/cos 공식으로 값을 하드코딩합니다. 임의의 길이로 자연스럽게 확장되고, 내적이 상대 거리 f(m−n)의 함수가 되는 성질을 가집니다. 원조 트랜스포머(Vaswani et al., 2017)의 기본값이었습니다.

ALiBi — 선형 편향(Attention with Linear Biases) 외부

위치 임베딩을 아예 더하지 않고, 대신 쿼리-키 점수에 거리에 비례하는 편향(bias)을 직접 빼줍니다. 슬라이드는 T5의 학습형 버킷 편향(bias(m,n)=βbucket(m−n), 원본 Raffel et al.)과, 더 단순하고 결정론적인 ALiBi(bias(m,n)=μ×(n−m))를 나란히 보여줍니다.

softmax( ⟨qm, kn⟩ ∕ √dk + bias(m,n) )

ALiBi의 강점은 "Train Short, Test Long"이라는 논문 제목 그대로, 짧은 길이로 학습해도 훨씬 긴 시퀀스로 잘 일반화(외삽)된다는 점입니다 — 헤드마다 기울기 μ만 다르게 고정하면 되므로 학습 파라미터가 전혀 늘지 않습니다.

RoPE — 회전 위치 임베딩(Rotary Position Embedding) 원본

오늘날 가장 널리 쓰이는 기본 선택입니다. 쿼리·키 벡터를 위치에 비례하는 각도만큼 회전시킵니다. 아래 3)절에서 자세히 다룹니다.

지금의 기본값: RoPE — 회전으로 상대 위치를 표현하다

RoPE(RoFormer: Enhanced Transformer with Rotary Position Embedding, Su et al., 2021)의 아이디어는 원본 "쿼리와 키 벡터를 회전 행렬로 돌리자"는 것입니다. 마치 시계 바늘을 위치 m만큼 미리 돌려놓고 비교하는 것과 같습니다. 2차원 벡터라면 각도 mθ만큼 회전시키는 다음 행렬을 곱합니다.

Rθ,m = ( cos(mθ)  −sin(mθ) ;  sin(mθ)  cos(mθ) ) 2×2 회전 행렬. d>2 차원에서는 이런 2×2 블록을 대각선으로 쌓아 전체 차원을 회전시킵니다 (block₁, block₂, …, blockd/2).

원본 회전 행렬을 곱한 뒤 내적을 계산하면, 놀랍게도 절대 위치 m, n은 사라지고 상대 거리 n−m만 남습니다.

qm knT = xm Wq Rθ,n−m WkT xnT 회전 두 번(위치 m, 위치 n)을 합성하면 회전각이 (n−m)θ 하나로 정리되기 때문입니다. 사인 함수 방식이 "내적이 f(m−n)"이라는 성질을 가졌던 것과 같은 계보의 결과입니다.

원본 또한 회전 각도가 멀어질수록 어텐션 가중치 상한이 점차 감쇠하는 경향("Attention weight has a long-term decay")을 보여, 먼 토큰일수록 자연스럽게 덜 주목하게 되는 유용한 부작용도 있습니다.

위젯 · RoPE 회전 시각화 종합·해석

쿼리 벡터 q(위치 m)와 키 벡터 k(위치 n)를 각각 mθ, nθ만큼 회전시켜 봅니다. 두 슬라이더를 움직여 내적이 실제로 (m−n)에만 의존하는지 직접 확인해 보세요.

03 — Mechanism variant

층마다 다시 맞추는 신호 크기: 레이어 정규화란 무엇인가 Layer Normalization

회의가 길어지면 발언자마다 목소리 크기가 들쭉날쭉해지고, 사회자는 중간중간 마이크 볼륨을 다시 맞춰줘야 대화가 이어집니다. 레이어 정규화(Layer Normalization, LN)가 트랜스포머 안에서 하는 일이 정확히 이것입니다 — 매 서브층을 통과한 뒤 값의 크기(스케일)를 다시 맞춰 다음 층으로 넘깁니다.

LN(x) = γ x̂ + β,   x̂ = (x − μ) ∕ √(σ² + ε) μ, σ²는 한 벡터 안의 모든 차원에 대한 평균·분산입니다. 평균을 빼고 표준편차로 나눠 신호를 표준화한 뒤, 학습 가능한 γ(스케일)·β(이동)로 다시 미세조정합니다.

원본 효과는 "Helps with training stability and convergence" — 학습을 더 안정적으로, 더 빠르게 수렴하도록 돕습니다(Ba et al., 2016).

어디에 놓을 것인가: Post-Norm vs Pre-Norm

Post-Norm 원본

Output = LayerNorm( x + SubLayer(x) )

잔차 연결(residual connection) 이후에 정규화를 적용합니다. 원조 트랜스포머(2017)의 기본값이었지만, 층이 깊어질수록 출력층 근처의 그래디언트가 커져 학습률 워밍업(warm-up)이 사실상 필수가 되는 단점이 있습니다(Xiong et al., 2020 외부).

Pre-Norm 원본

Output = x + SubLayer( LayerNorm(x) )

서브층에 들어가기 이전에 정규화를 적용합니다. Xiong et al.(2020, 외부)은 평균장 이론(mean field theory)으로 Pre-Norm이 초기화 시점의 그래디언트를 안정시켜 워밍업 없이도 학습이 잘 된다는 것을 보였습니다. 오늘날 대부분의 대형 언어모델이 이 방식을 기본으로 채택합니다.

Pre-Norm + RMSNorm 원본

γ x̂ + β  ⟶  γ · x ∕ RMS(x)

RMSNorm(Root Mean Square Layer Normalization, Zhang & Sennrich, 2019 외부)은 평균을 빼는 재중심화(re-centering) 단계를 아예 생략하고 크기(스케일)만 맞춥니다. 계산량이 줄어 학습·추론이 빨라지면서도 성능은 거의 그대로 유지됩니다. 원본 슬라이드는 "Pre-Norm + RMSNorm"을 오늘날의 기본 조합으로 명시합니다.

04 — Bottlenecks & dynamics

왜 모든 토큰이 서로를 다 쳐다볼 필요는 없을까? Sparse Attention & Head Sharing

동창회에서 참석자 전원과 한 명씩 대화하려면 인원이 늘수록 대화 조합이 기하급수적으로 늘어납니다. 다중 헤드 어텐션(Multi-Head Attention, MHA)의 softmax(QKT/√dk)V 계산도 마찬가지로, 토큰 수 n에 대해 O(n²)으로 비용이 늘어납니다.

원본 지난 강의 복습: MHA
h개의 어텐션 헤드가 각자 독립적인 Q/K/V 선형변환을 갖고 병렬로 어텐션을 계산한 뒤, 결과를 이어붙여(Concat) 다시 선형변환합니다(Vaswani et al., 2017). 헤드 수만큼 "서로 다른 관점의 서기"를 두는 셈입니다.

1) 다 보는 대신 일부만 보기: 희소·슬라이딩 윈도우 어텐션

원본 Longformer(Beltagy et al., 2020)는 모든 토큰 쌍을 계산하는 대신, [CLS]처럼 중요한 토큰에는 전역(global) 어텐션을, 나머지 토큰에는 이웃 몇 개만 보는 지역(local) 어텐션을 섞습니다. 마치 큰 강연장에서 사회자(전역)만 모두와 눈을 맞추고, 청중(지역)은 옆자리 몇 명하고만 소곤대는 것과 같습니다.

슬라이딩 윈도우 어텐션(Sliding Window Attention, SWA)도 같은 계보입니다. 원본 컴퓨터 비전의 합성곱(convolution) 수용 영역(receptive field)과 닮았다고 슬라이드는 설명합니다 — 층을 쌓을수록 실제로 참조하는 범위가 넓어지기 때문입니다. Mistral 7B(Jiang et al., 2023 외부)는 창 크기 4096 토큰의 SWA로 8K 문맥을 처리합니다.

2) 서기를 몇 명 쓸까: MHA → GQA → MQA

원본 두 번째 절감 전략은 키·값 헤드를 쿼리 헤드끼리 공유하는 것입니다. 쿼리 헤드 h개를 G개의 그룹으로 묶고, 그룹 안에서는 하나의 K/V 헤드를 함께 씁니다. 마치 발언자 h명에 서기를 몇 명 배정할지 정하는 문제와 같습니다.

G = h → MHA

Multi-Head Attention(다중 헤드 어텐션). 발언자마다 전담 서기. 품질은 가장 좋지만 K/V 캐시가 가장 무겁습니다.

1 < G < h → GQA

Grouped-Query Attention(그룹 쿼리 어텐션). 몇 개 조로 나눠 서기를 배정. 속도와 품질의 절충점 — 오늘날 실무의 기본값입니다.

G = 1 → MQA

Multi-Query Attention(다중 쿼리 어텐션). 서기 단 한 명이 전원을 기록. 가장 저렴하지만 품질 손실이 가장 큽니다.

슬라이드가 이 3분류를 이름으로 명시적으로 소개합니다(원본). MQA의 원조 논문은 Shazeer(2019, 외부) "Fast Transformer Decoding: One Write-Head is All You Need"이며, GQA의 원조 논문은 Ainslie et al.(2023, 외부) "GQA: Training Generalized Multi-Query Transformer Models from Multi-Head Checkpoints"입니다.

긴 문맥 어텐션의 병목을 하나씩 풀어보기

「해결 →」 버튼을 눌러가며, 하나의 병목이 풀릴 때 다음 병목이 어떻게 드러나는지 따라가 봅시다.

위젯 · KV 캐시 메모리 계산기 종합·해석

디코딩 중에는 이미 계산한 키·값을 KV 캐시에 저장해 재사용합니다. 캐시 크기는 K/V 헤드 수 G에 비례하므로, G를 줄이면(MQA·GQA) 캐시도 그만큼 줄어듭니다. 기본값은 Mistral 7B(L=32, H=4096, h=32, G=8, Jiang et al., 2023 외부)의 실제 설정입니다.

05 — Alternatives

인코더, 디코더, 그리고 그 사이: 트랜스포머 계열 3분류 Encoder-Decoder / Encoder-only / Decoder-only

원본 원조 트랜스포머는 인코더와 디코더를 모두 갖췄지만, 이후 등장한 모델들은 목적에 따라 한쪽만 남기는 방향으로 갈라졌습니다. 슬라이드는 이를 3가지 범주로 정리합니다.

인코더-디코더 (Encoder-Decoder) 원본

텍스트를 텍스트로 변환하는(text-to-text) 과제에 적합합니다. 대표 모델: T5, mT5, ByT5. 원조 구조를 그대로 유지하며, 번역·요약처럼 입력과 출력이 모두 자연어인 과제에 강합니다. 원본 2018~2022년 사이 인코더를 포함한 계열(인코더-디코더 + 인코더-온리)이 특히 인기 있었습니다.

인코더-온리 (Encoder-only) 원본

디코더를 떼어내고 인코더가 만든 임베딩을 클래스 예측용으로 투영합니다(예: 감정 분석). 대표 모델: BERT, DistilBERT, RoBERTa. 아래 06절에서 깊게 다룹니다.

디코더-온리 (Decoder-only) 원본

인코더 없이 디코더만으로 텍스트를 생성합니다. 대표 모델: GPT 계열. 원본 슬라이드는 이 계열을 "지금 가장 인기 있는(Popular now!)" 범주로 표시합니다 — 2025년 강의 시점 기준의 관찰이며, 다음 강의(3강)에서 LLM으로 본격적으로 다룹니다.

06 — Deep dive

왜 하필 이름이 BERT일까? BERT Deep Dive

원본 BERT는 Bidirectional Encoder Representations from Transformers(트랜스포머로부터의 양방향 인코더 표현)의 약자입니다. 이름 하나하나가 설계 선택을 담고 있습니다.

Bidirectional(양방향) 원본

마스킹 없는 어텐션이라 모든 토큰이 문장 전체(앞과 뒤 모두)를 봅니다. 마스킹된 디코더 어텐션은 미래 토큰을 가리므로 "Not bidirectional"입니다 — 이것이 GPT 계열과의 근본적 차이입니다.

Encoder(인코더) 원본

원조 트랜스포머의 인코더 스택만 남기고 디코더는 통째로 제거합니다. 목표는 텍스트 생성이 아니라 분류에 쓸 좋은 표현(representation)을 얻는 것입니다.

이름 짓기 유머로, ELMo(Embeddings from Language Models, Peters et al., 2018 외부)에 이어 세서미 스트리트 캐릭터 이름을 딴 것으로 알려져 있습니다 원본.

입력을 준비하는 법: WordPiece와 세 겹의 임베딩

원본 BERT는 약 3만 개 어휘의 WordPiece 알고리즘으로 토큰화하고, 문장 앞에 [CLS], 문장 구분에 [SEP], 마스킹에 [MASK] 토큰을 사용합니다. 최종 입력 벡터는 세 임베딩의 합입니다.

토큰 임베딩

어휘마다 학습된 거대한 조회 테이블(lookup table).

위치 임베딩

02절에서 다룬 방식(학습형 또는 고정 사인함수).

(신규!) 세그먼트 임베딩

문장 A/B 중 어디에 속하는지 표시하는 공유 임베딩.

사전학습 대리 과제 두 가지: MLM과 NSP

MLM — Masked Language Modeling 원본

입력 토큰의 15%를 예측 대상으로 삼되, 그중 80%는 [MASK]로 바꾸고, 10%는 임의의 다른 단어로, 10%는 그대로 둡니다. 문맥으로부터 언어를 배우게 하는 동시에, 언어의 확률적 성격을 반영한 일종의 정규화 효과가 있습니다.

NSP — Next Sentence Prediction 원본

두 문장을 뽑아 50%는 실제로 이어지는 쌍, 50%는 무작위 쌍으로 구성하고 이어지는지 여부를 맞히게 합니다. 레이블 없이도 문맥 이해 능력을 학습시키는 손쉬운 분류 과제입니다.

모델 크기와 하이퍼파라미터

원본 BERT의 크기는 레이어 수 L, 은닉 차원 H, 헤드 수 A로 정해집니다.

파인튜닝 실습: 감정 추출 걸어가 보기

원본 슬라이드는 "This teddy bear is SO CUTE!"라는 문장으로 감정 추출(sentiment extraction) 파인튜닝 과정을 단계별로 보여줍니다. 소문자화 → 토큰화 → [CLS]/[SEP]/[PAD] 삽입 → (토큰+위치+세그먼트) 임베딩 합산 → 사전학습된 인코더 통과 → [CLS] 위치 출력만 작은 FFN(Feed-Forward Network)에 통과시켜 이진 분류 결과를 얻습니다.

종합·해석 핵심 트릭
파인튜닝(Fine-tuning) 단계는 이미 대규모로 사전학습된 가중치에서 출발하고, 초기 층을 동결(freezing)하면 복잡도-성능 트레이드오프가 개선되기도 합니다. 사전학습 분포·목표와 가까운 과제일수록 적은 레이블 데이터로도 좋은 성능을 냅니다. — 원본 슬라이드 요약

장단점과 트릭: 지식 증류, DistilBERT, RoBERTa

장점 원본

최고 수준 성능, 진정한 문맥적 표현에 가까움, 다양한 분류 과제에 적응 가능. 인코딩이 필요한 산업 전반에 널리 쓰입니다.

한계 원본

문맥 창 크기 제한, 연산 비용이 커 저지연·비용 민감 서비스에 부담, MLM/NSP 사전학습 후 파인튜닝이라는 복잡한 2단계 학습 파이프라인이 필요합니다.

원본 지식 증류(Knowledge Distillation)는 이 비용 문제를 줄이는 트릭입니다. "The soft targets contain almost all the knowledge."(Hinton, 2014년 발표 "Dark Knowledge"; 정식 논문은 Hinton, Vinyals & Dean, Distilling the Knowledge in a Neural Network, 2015 외부) — 크고 정확한 교사(Teacher) 모델의 부드러운 출력 분포를 작은 학생(Student) 모델이 흉내내도록 KL 발산으로 학습시킵니다.

KL(ŷT ‖ ŷS) = Σi ŷT(i) log( ŷT(i) ∕ ŷS(i) ) 교사 분포 ŷT와 학생 분포 ŷS가 얼마나 다른지 측정해, 학생이 교사의 "부드러운" 확신 패턴까지 배우게 합니다.

DistilBERT 원본

BERT-Base(12층)를 6층으로 증류해 약 1.6배 빠르면서 성능은 약 97% 유지합니다(Sanh et al., 2019 외부).

RoBERTa 원본

NSP/세그먼트 인코딩 제거(영향 거의 없음), 정적 → 동적 마스킹, 16GB → 160GB 데이터, 훨씬 긴 학습으로 같은 아키텍처에서 벤치마크 +4%를 달성했습니다(Liu et al., 2019 외부).
